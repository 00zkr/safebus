# Documents and Papers to Prepare Next

This file lists the useful documents you can create from the SafeBus project.

## Core delivery documents

1. `README.md` in English

   Purpose: explain the project quickly for a teacher, evaluator, developer, or jury.

   Suggested sections:

   - Project title
   - Short description
   - Features
   - Technologies
   - Architecture
   - Installation
   - Demo accounts
   - API summary
   - Screenshots
   - Future improvements

2. English PowerPoint presentation

   Purpose: present the project clearly during a defense or demo.

   Suggested length: 12 to 15 slides.

   Suggested title:

   ```text
   SafeBus: School Transport Tracking System
   ```

3. French LaTeX report, French academic format

   Purpose: final formal report in French.

   Suggested title:

   ```text
   SafeBus : Systeme de suivi du transport scolaire
   ```

   Suggested chapters:

   - Introduction generale
   - Contexte du projet
   - Analyse des besoins
   - Conception
   - Realisation
   - Tests et validation
   - Deploiement
   - Conclusion generale

4. English LaTeX report, English academic format

   Purpose: international or technical version of the final report.

   Suggested title:

   ```text
   SafeBus: School Transport Tracking System
   ```

   Suggested chapters:

   - General introduction
   - Project context
   - Requirements analysis
   - System design
   - Implementation
   - Testing and validation
   - Deployment
   - Conclusion

## Technical papers

1. Software Requirements Specification

   Include:

   - Purpose
   - Scope
   - Actors
   - Functional requirements
   - Non-functional requirements
   - Constraints
   - Use cases

2. Software Design Document

   Include:

   - Global architecture
   - Component design
   - Backend design
   - Web design
   - Mobile design
   - Data model
   - Security model

3. API Documentation

   Include:

   - Authentication endpoint
   - Student endpoints
   - Bus endpoints
   - Route endpoints
   - Notification endpoints
   - Driver endpoints
   - Demo reset endpoint
   - Request and response examples

4. Database Documentation

   Include:

   - ERD
   - Table dictionary
   - Relationships
   - Seed data explanation
   - SQL scripts

5. Test Plan and Test Report

   Include:

   - Test strategy
   - Test environment
   - Test cases
   - Results
   - Screenshots
   - Bugs found
   - Limitations

6. Deployment Guide

   Include:

   - Prerequisites
   - Docker Compose setup
   - Environment variables
   - Run commands
   - Ports
   - Troubleshooting

7. User Manual

   Create one manual for each role:

   - Administrator manual
   - Driver manual
   - Parent manual

8. Maintenance Guide

   Include:

   - Project structure
   - How to add a new API endpoint
   - How to add a dashboard page
   - How to add a mobile screen
   - How to update database schema
   - How to reset demo data

## Academic support papers

1. Abstract in French and English

   Prepare short abstracts of 150 to 250 words.

2. Resume / Summary

   Prepare a one-page project summary for quick evaluation.

3. Problem Statement

   Explain the school transport communication problem and why tracking is useful.

4. State of the Art

   Compare SafeBus with existing transport tracking or fleet management solutions.

5. UML Diagrams

   Useful diagrams:

   - Use case diagram
   - Class diagram
   - Sequence diagram for login
   - Sequence diagram for driver status update
   - Sequence diagram for parent notification
   - Deployment diagram

6. Gantt Chart

   Show the planning of the project development phases.

7. Risk Management Sheet

   Include:

   - Technical risks
   - Planning risks
   - Security risks
   - Testing risks
   - Mitigation actions

8. Glossary

   Define terms such as API, JWT, REST, Socket.IO, Docker, MySQL, ViewModel, and realtime event.
